# LLVM AI Tool Attribution Guide

This skill guides proper AI attribution when contributing to the LLVM project (llvm/llvm-project).

## LLVM AI Policy Summary

LLVM has an official AI Tool Use Policy ([AIToolPolicy.html](https://llvm.org/docs/AIToolPolicy.html)) with these key requirements:

| Requirement | Details |
|-------------|---------|
| **Human in the Loop** | Contributors MUST read and review all LLM-generated code/text before asking for review |
| **Accountability** | The contributor is always the author and fully accountable for their contributions |
| **Transparency** | Label contributions with substantial AI-generated content |
| **No Autonomous Agents** | Banned: agents that take action without human approval (e.g., GitHub `@claude` agent) |
| **Good First Issue Ban** | **FORBIDDEN**: Using AI to fix issues labeled `good first issue` |

## Attribution Formats

When AI tools assisted the contribution, use one of these trailer formats in commit messages or PR descriptions:

### Claude Attribution
```
Assisted-by: Claude Code
Assisted-by: claude-4.6-opus
Assisted-by: Claude Sonnet 4.6
Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

### Copilot Attribution
```
Assisted-by: GitHub Copilot
Co-authored-by: Copilot <175728472+Copilot@users.noreply.github.com>
Co-authored-by: Copilot Autofix powered by AI
```

### ChatGPT Attribution
```
Assisted-by: ChatGPT
Assisted-by: ChatGPT (refine implementation + tests)
```

### General AI Disclosure
```
AI Usage Disclosure: AI tools (Claude Sonnet 4.5) were used to assist with implementation
This patch was generated by AI, but I reviewed it
While the test cases were generated by AI, I've personally checked every single line
```

## Workflow

### Before Submitting

1. **Confirm AI involvement**: Ask the user which AI tools were used
   ```
   "Before we submit this to llvm-project, I need to confirm the attribution.
   Which AI tools assisted you in this work? (e.g., Claude, Copilot, ChatGPT)"
   ```

2. **Verify human review**: Ensure the user has reviewed and understands all AI-generated content

3. **Check policy compliance**:
   - Is this fixing a `good first issue`? → **STOP, this is forbidden**
   - Is this an autonomous agent action? → **STOP, banned by policy**

### Preparing Commit/PR Message

Add appropriate attribution trailer based on AI tools used:

**Example commit message:**
```
[MLIR][SparseTensor] Fix direct op erasure in sparse tensor operations

The previous implementation had incorrect handling of direct op erasure
which caused crashes in certain sparse tensor patterns.

Assisted-by: Claude Sonnet 4.6
```

**Example PR description:**
```
## Summary
Fixes incorrect handling of direct op erasure in sparse tensor operations.

## Implementation
- Updated the erasure logic to properly track operation dependencies
- Added test cases for the edge cases that were causing crashes

## AI Usage
This implementation was assisted by Claude Code. I have reviewed all
generated code and tested it locally.

Assisted-by: Claude Code
```

## Policy Violation Handling

If a maintainer judges a contribution violates the policy, they may respond with:

> This PR doesn't appear to comply with our policy on tool-generated content,
> and requires additional justification for why it is valuable enough to the
> project for us to review it. Please see our developer policy on
> AI-generated contributions: http://llvm.org/docs/AIToolPolicy.html

If this happens, help the user:
1. Reduce the change size/complexity
2. Increase usefulness to the community
3. Provide stronger justification for human review value
