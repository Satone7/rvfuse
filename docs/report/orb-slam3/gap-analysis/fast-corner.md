## Cross-Platform Comparison

### Integer Pixel Threshold Comparison

| Platform | Instruction(s) | Count | Elements/Op | Throughput (elements/cycle @ native width) |
|----------|---------------|-------|-------------|-------------------------------------------|
| **RVV512** | `vmslt.vx` + `vmsgt.vx` | 2 | 64 (u8) | 32 |
| **x86 AVX2** | `_mm256_subs_epu8` + `_mm256_cmpeq_epi8` | 2 | 32 (u8) | 16 |
| **x86 AVX-512** | `vpsubusb` + `vpcmpeqb` | 2 | 64 (u8) | 32 |
| **ARM NEON** | `vqsub.u8` + `vceq.u8` | 2 | 16 (u8) | 8 |
| **ARM SVE** | `sub` + `cmpeq` (predicated) | 2 | VL | VL/2 |
| **LoongArch LSX** | `vsub.b` + `vseq.b` | 2 | 16 (u8) | 8 |

**RVV advantage**: Matches AVX-512 at 64 elements. 2× NEON/LSX throughput. Mask registers (vmslt/vmsgt) are architecturally separate from data registers — reduces register pressure vs x86 where masks share 256/512-bit data registers.

### Mask-Based Corner Selection

The FAST algorithm requires ≥9 contiguous brighter-or-darker pixels. RVV mask registers enable:

```
// RVV approach: generate masks then count contiguous
vmslt.vx v0, v8, a2        // 64-bit mask: 1=dark, 0=not-dark
// Scan mask for ≥9 contiguous set bits
vcpop.m  a0, v0             // Population count of mask
```

| Platform | Mask Width | Contiguous Detection Method |
|----------|-----------|---------------------------|
| **RVV512** | 64 bits | `vcpop.m` for popcount + mask shift for contiguity |
| **x86 AVX2** | 32 bits | `_mm256_movemask_epi8` (32-bit int mask) + scalar loop |
| **x86 AVX-512** | 64 bits | `vpcmpb` + `kmov` to GPR + scalar BSF |
| **ARM NEON** | 16 bits | `vget_lane` per-lane extraction |
| **ARM SVE** | VL bits | `whilelo` + `brka` for contiguity |

**RVV advantage**: `vcpop.m` counts mask bits in hardware (1 instruction). x86 requires kmov + popcnt (GPR). NEON has no native popcount on vector masks.

## Gaps Identified

1. **No RVV mask-to-index instruction**: After `vmslt` produces a mask of corner candidates, extracting the indices of set bits requires a scalar loop. x86 has `tzcnt`/`blsi` for fast bit iteration. An RVV `vfirst.m` (find-first-set on mask) or `vcompress` with index output would enable fully vectorized corner position extraction.

2. **Contiguous run detection**: The ≥9 contiguous pixel requirement currently falls to scalar code. AVX2/AVX-512 use `_mm256_movemask_epi8` to extract a 32-bit integer mask and test contiguity with bit operations (`(mask & (mask>>1) & ... & (mask>>8))`). RVV has no efficient way to test contiguous mask bits — `vcpop.m` gives total count but not contiguity.

3. **FAST score computation**: Corner score = sum(|I_circle - I_p|) across all 16 circle pixels. RVV `vsaddu` (sum of absolute differences) would be ideal here but is not generated by the auto-vectorizer. x86 uses `_mm_sad_epu8` (PSADBW) which computes SAD in hardware.
